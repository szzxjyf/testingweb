# Tariff 异常交易数据处理流程说明

---

## 一、整体流程概览

```
XLSX 原始文件（若干个）          CSV 原始文件（若干个）
        │                              │
   ① 逐个筛选异常                 ② 逐个筛选异常
        │                              │
   各自输出 _abnormal.csv          各自输出 _abnormal.csv
        │                              │
   ③ 合并为一个文件               ④ 合并为一个文件
   （xlsx_merged_all.csv）        （csv_merged_all.csv）
        │                              │
        └──────────┬───────────────────┘
                   │
            ⑤ 统一字段名，合并为一个总表
              （final_combined.csv）
                   │
            ⑥ 按规则分组拆分
                   │
        ┌──────┬───┴──────┐
      FCY.csv LCY.csv AC.csv 
```

---

## 二、各步骤详细说明

### 步骤 ①② — 异常交易筛选（xlsx.py / csv.py）

对每个原始数据文件，执行以下筛选逻辑：

#### 第一步：缩小数据范围

从全量数据中，仅保留符合以下条件的记录：

| 条件 | 说明 |
|------|------|
| 条件 A | OWNER（负责部门）= GLCM 或 GPS |
| 条件 B | OWNER 为空，且客户所属条线 = CMB 或 CIBM，且损益科目代码 ≠ CMB_NC1 / CMB_NCC |
| 条件 C | 交易附言1 ≠ "VAT PAYABLE - OUTPUT VAT" |

**最终保留**：满足（条件A **或** 条件B）**且** 满足条件C 的记录。

#### 第二步：对保留的数据，按规则判定异常

以下规则按顺序执行，命中任一规则即标记为异常：

**规则一：FCY_ALL（外币-全量）**

损益科目代码属于以下之一的，**全部标记为异常**：
- CMR_DDI、CMR_BCC、CIE_TT

**规则二：AC_ALL（账户-全量）**

损益科目代码属于以下之一的，**全部标记为异常**：
- CMR_GEN、SVC_CUI、SVC_SSV、SVC_SS3、SVC_RSC
- CMR_CUI、CMR_SSV、CMR_SS3、CMR_ADT、CMR_AD1、CMR_PY2

**规则三：LCY（本币-金额阈值）**

损益科目代码匹配以下之一，且金额 **< 0 或 > 阈值** 的，标记为异常：

| 损益科目代码 | 阈值（人民币） |
|-------------|---------------|
| CMR_EFT | 200 |
| CMR_APO | 200 |
| CMR_RM1 | 200 |

**规则四：FCY（外币-金额阈值）**

损益科目代码匹配以下之一，且金额 **< 0 或 > 阈值** 的，标记为异常：

| 损益科目代码 | 阈值（人民币） |
|-------------|---------------|
| CMR_TT | 1,000 |
| SND_TT | 200 |
| CBL_TT | 140 |
| CMR_TTI | 55 |
| CMR_DMX | 500,000 |

**规则五：OTHERS（其他）**

损益科目代码 **不在** 以上任何规则涉及的代码范围内的，**全部标记为异常**。

---

### 步骤 ③④ — 各自合并（merge.py）

将 XLSX 版输出的多个 `_abnormal.csv` 文件合并为一个 `merged_all.csv`。

将 CSV  版输出的多个 `_abnormal.csv` 文件合并为一个 `merged_all.csv`。

每行数据会附加 `__source_file` 字段，标注其来自哪个原始文件，便于追溯。

---

### 步骤 ⑤ — 统一字段名并合并总表（combine.py）

由于 XLSX 版使用中文列名、CSV 版使用英文列名，需要统一后才能合并。

统一后的列名格式为 **"中文(英文)"**，完整对照如下：

| XLSX 原列名 | CSV 原列名 | 统一后列名 |
|------------|-----------|-----------|
| 客户代码 | cust_no | 客户代码(cust_no) |
| 客户英文名 | cust_name_en | 客户英文名(cust_name_en) |
| 客户中文名 | cust_name_cn | 客户中文名(cust_name_cn) |
| 客户类型 | cust_family | 客户类型(cust_family) |
| 交易附言1 | trans_remark1 | 交易附言1(trans_remark1) |
| 交易附言2 | trans_remark2 | 交易附言2(trans_remark2) |
| 交易附言3 | trans_remark3 | 交易附言3(trans_remark3) |
| 交易附言4 | trans_remark4 | 交易附言4(trans_remark4) |
| 入账日期 | data_dt | 入账日期(data_dt) |
| 交易币种 | ccy_code | 交易币种(ccy_code) |
| 交易金额 | trans_amt | 交易金额(trans_amt) |
| 人民币等值金额 | trans_amt_cny | 人民币等值金额(trans_amt_cny) |
| 客户所属业务条线 | cust_group | 客户所属业务条线(cust_group) |
| 交易日期 | trans_date | 交易日期(trans_date) |
| 关联账户账号 | asso_acct_no | 关联账户账号(asso_acct_no) |
| 收费项目 | tariff_type | 收费项目(tariff_type) |
| OWNER | charge_department | OWNER(charge_department) |
| VAT_RELATED | vat_related | VAT_RELATED(vat_related) |
| NON_TARIFF | non_tariff | NON_TARIFF(non_tariff) |
| 月份 | trans_month | 月份(trans_month) |
| GMIS | gmis_code | GMIS(gmis_code) |
| 损益科目代码 | 损益科目代码 | 损益科目代码(ie_item+ie_type拼接) |

- 各自独有的字段（如 XLSX 的"序列号"、CSV 的"branch_name"等）原样保留，对方无数据的行显示为空。
- 每行数据附加 `__data_source` 字段，标注来源为 XLSX 或 CSV。

合并后输出为 `final_combined.csv`。

---

### 步骤 ⑥ — 按规则分组拆分（split.py）

将 `final_combined.csv` 按照异常规则类型（`__rule_group`）拆分为独立文件：

| 输出文件 | 包含的规则分组 | 说明 |
|---------|-------------|------|
| **FCY.csv** | FCY + FCY_ALL | 所有外币相关的异常记录 |
| **LCY.csv** | LCY | 所有本币相关的异常记录 |
| **AC.csv** | AC_ALL | 所有账户类的异常记录 |
| **OTHERS.csv** | OTHERS | 不在上述规则范围内的异常记录 |

---

## 三、脚本执行顺序

| 顺序 | 脚本 | 作用 |
|------|------|------|
| 1 | `xlsx.py` | 处理 XLSX 原始文件，筛选异常并导出 |
| 2 | `csv.py` | 处理 CSV 原始文件，筛选异常并导出 |
| 3 | `merge.py` | 将各自的多个异常文件合并为一个 |
| 4 | `combine.py` | 统一字段名，合并 XLSX 版和 CSV 版为总表 |
| 5 | `split.py` | 按规则分组拆分为 FCY / LCY / AC / OTHERS |
